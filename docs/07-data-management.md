# 07 — وحدة إدارة البيانات (Data Management Module)

> Module مستقل — يُضاف بعد ما البرنامج الأساسي شغّال. لا يلمس الـ Modules التانية.

## الهيكل

```
app/Modules/DataManagement/
├── ExportService.php          ← تصدير أي Model لـ Excel أو PDF
├── ImportService.php          ← استيراد من Excel مع validation
├── PeriodRollbackService.php  ← إلغاء مدخلات فترة زمنية
└── DataCleanupService.php     ← حذف بيانات محددة بأمان

app/Filament/Pages/
├── ExportCenterPage.php
├── ImportCenterPage.php
├── PeriodManagerPage.php
└── DataCleanupPage.php

app/Jobs/
├── ExportDataJob.php          ← تصدير كبير في Background
└── ImportDataJob.php          ← استيراد كبير في Background
```

## 1. مركز التصدير (Export Center)

### ما يُصدّر:

**بيانات أساسية:**
- قائمة العملاء كاملة
- قائمة الموردين
- قائمة الأصناف والأسعار
- قائمة الشركات (المصنّعين)
- قائمة أسعار لإصدار معين

**بيانات مالية:**
- الفواتير في فترة معينة
- سندات القبض والصرف
- الشيكات الواردة والصادرة
- حركة خزينة لفترة معينة

**تقارير:**
- تقرير الأرباح والخسائر
- أعمار الديون
- رصيد المخزون الحالي
- الأصناف الراكدة

### الصيغ المدعومة:
- Excel (.xlsx) — عبر Maatwebsite/Excel
- CSV
- PDF — عبر DomPDF

### التنفيذ:
- البيانات الصغيرة (< 1000 صف) → تصدير فوري
- البيانات الكبيرة (> 1000 صف) → ExportDataJob في Queue → إشعار عند الانتهاء
- كل عملية تصدير تُسجّل في سجل (من صدّر، متى، ماذا)

## 2. مركز الاستيراد (Import Center)

### ما يُستورد:
- عملاء من Excel
- موردين من Excel
- أصناف (منتجات) من Excel
- قوائم أسعار من Excel

### Workflow الاستيراد:
```
1. رفع ملف Excel
2. تحليل الأعمدة وربطها بالحقول (Column Mapping)
3. Validation — التحقق من كل صف
4. Preview — عرض البيانات قبل الحفظ مع تلوين الأخطاء
5. تأكيد → حفظ في قاعدة البيانات
6. سجل الاستيراد (عدد المدخل، عدد المرفوض، الأخطاء)
```

### قواعد الاستيراد:
- **Validation قبل الحفظ** — أي صف فيه خطأ يُرفض ويُعرض السبب
- **Diff Viewer** — لو البيانات موجودة: يعرض الفروقات ويسأل: تحديث أم تجاهل؟
- **Undo** — تراجع عن آخر استيراد (يحذف كل ما أُدخل في هذه الدفعة)
- البيانات الكبيرة → ImportDataJob في Queue

## 3. مدير الفترات (Period Manager)

### الوظائف:
- **قفل فترة مالية** — يمنع أي معاملة جديدة فيها
- **فتح فترة مقفولة** — بصلاحية خاصة (`accounting.lock_period`)
- **ترحيل أرصدة ختامية** — نقل الأرصدة لأول الفترة الجديدة
- **Rollback** — إلغاء كل مدخلات فترة زمنية (حالة طوارئ)

### Rollback بالتفصيل:
```
1. تحديد الفترة الزمنية (من تاريخ — إلى تاريخ)
2. عرض كل المعاملات في الفترة (فواتير، مقبوضات، مدفوعات، قيود)
3. تأكيد مزدوج من المستخدم
4. DB::transaction() →
   - حذف (soft delete) كل الفواتير في الفترة
   - عكس حركات المخزون المرتبطة
   - عكس حركات الخزينة المرتبطة
   - حذف القيود المحاسبية المرتبطة
   - إعادة حساب الأرصدة
5. تسجيل عملية الـ Rollback في سجل منفصل
```

### تحذيرات:
- Rollback عملية خطيرة — تحتاج تأكيد مزدوج + كلمة سر
- يجب أن تُنفذ من Super Admin فقط
- السجل الأصلي يبقى (soft deleted) للمراجعة

## 4. أدوات تنظيف البيانات (Data Cleanup)

### الأدوات:

**اكتشاف التكرارات:**
- عملاء بنفس الاسم أو التليفون
- منتجات بنفس الاسم أو الكود
- عرض المكررات مع خيار الدمج

**دمج السجلات:**
- اختيار السجل الأساسي + السجل المكرر
- نقل كل المعاملات من المكرر للأساسي
- أرشفة المكرر (soft delete)

**أرشفة البيانات غير النشطة:**
- منتجات بدون حركة من X شهر
- عملاء بدون معاملات من X شهر
- نقلها لحالة "أرشيف" — تختفي من البحث العادي لكن تبقى في الداتابيز

### المبدأ الحاكم:
> **أرشفة لا حذف. أي "حذف" في هذه الوحدة = soft delete + تسجيل في السجل.**
